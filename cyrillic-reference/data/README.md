# `data/` — machine-readable JSON

Machine-readable counterparts to the Markdown tables in the parent directory. Generated by [`../tools/generate_json.py`](../tools/generate_json.py) — pure Python 3 stdlib, reads the committed `.md` tables as its source of truth.

## Contents

| File | Description |
| --- | --- |
| [`pan-cyrillic.json`](pan-cyrillic.json) | Every in-scope codepoint (351 entries), sorted by Unicode value (ascending hex), with description, decomposition, SVG/plotter paths, nested locl variants, and a `used_by[]` list of languages. |
| [`languages/<Name>.json`](languages/) | One file per in-scope language (77 total — `Kaitag` and `Uzbek` skipped). Meta block (BCP 47 / ISO / OT tags, Paratype identifiers, confidence grade) plus `characters[]` filtered to the language's codepoints, with locl variants attached where the language declares them. |

## `pan-cyrillic.json`

Every in-scope codepoint, sorted by Unicode value. Variants are nested under the base character. Shape:

```json
{
    "meta": {
        "description": "Pan-Cyrillic character reference across 77 languages. Sorted by Unicode codepoint (ascending hex). Variants nested under the base character.",
        "languages_in_scope": 77,
        "generated_by": "cyrillic-reference/tools/generate_json.py"
    },
    "characters": [
        {
            "character": "б",
            "codepoint": "0431",
            "decomposition": [],
            "case": "lowercase",
            "description": "Cyrillic Small Letter Be",
            "diagram": {
                "svg":     { "sans": "svg/Sans/lc/0431.svg",          "serif": "svg/Serif/lc/0431.svg" },
                "plotter": { "sans": "glyphplotter/Sans/lc/0431.txt", "serif": "glyphplotter/Serif/lc/0431.txt" }
            },
            "variants": [
                {
                    "kind": "locl",
                    "language_tag": "sr",
                    "feature_locl_tag": "SRB ",
                    "style": "any",
                    "diagram": {
                        "svg":     { "sans": "svg/Sans/variants/0431.sr.svg",          "serif": "svg/Serif/variants/0431.sr.svg" },
                        "plotter": { "sans": "glyphplotter/Sans/variants/0431.sr.txt", "serif": "glyphplotter/Serif/variants/0431.sr.txt" }
                    },
                    "used_by": ["Macedonian", "Serbian"]
                }
            ],
            "used_by": ["Abazin", "Abkhazian", "…"]
        }
    ]
}
```

## `languages/<Name>.json`

One file per in-scope language (77 total). `<Name>` is the language's `name_eng` with spaces replaced by underscores, parentheses preserved — e.g. [`Abkhazian.json`](languages/Abkhazian.json), [`Saami_Kildin.json`](languages/Saami_Kildin.json), [`Altaic_(Oirot).json`](<languages/Altaic_(Oirot).json>), [`Evenki_(Tungus).json`](<languages/Evenki_(Tungus).json>). `Kaitag` and `Uzbek` are listed in the source registry but excluded from this output.

```json
{
    "meta": {
        "name_eng": "Abkhazian",
        "name_rus": "Абхазский",
        "alternative_names": ["Bzyb", "Abzhui", "Samurzakan"],
        "code_pt": 90,
        "url": "https://paratype.github.io/cyrillic-languages/index.html?lang=Abkhazian&group=cyrillic&ui=en&pg=2",

        "default_language_tag": "ru",
        "feature_locl_tag": "ABK ",
        "iso639_1": "ab",
        "iso639_3": "abk",
        "confidence": "high",
        "note": null
    },
    "characters": [
        {
            "character": "А",
            "codepoint": "0410",
            "decomposition": [],
            "case": "uppercase",
            "description": "Cyrillic Capital Letter A",
            "categories": ["alphabet"],
            "diagram": {
```

## Field reference

### `meta` (per-language file only)

| field | source | semantics |
| --- | --- | --- |
| `name_eng`, `name_rus` | `cyrillic_library.json` | English and Russian display names. |
| `alternative_names` | `base/<Name>.json#alt_names_eng`, deduplicated against `name_eng` | Commonly-cited alternative labels. Array, possibly empty. |
| `code_pt` | `cyrillic_library.json` | Paratype's internal integer per-language identifier. |
| `url` | derived from `name_eng` | Canonical URL on the Paratype Cyrillic Languages reference site. Spaces are percent-encoded (`%20`); parentheses kept literal. |
| `language_tag` | `cyrillic_library.json#language_tag` | BCP 47 tag. Note: **Macedonian is temporarily `"sr"`** as a font-rendering workaround and will revert to `"mk"` once PT Expert ships `MKD` locl support. |
| `default_language_tag` | `base/<Name>.json#local` | OT locale used as the source file's default. One of `ru`, `bg`, `sr`, `ba`, `cv`. |
| `feature_locl_tag` | `language-tags.json#ot_lang` | 4-letter OpenType language-system tag (e.g. `ABK `, `BGR `, `SRB `), padded to 4 chars with trailing space per the OT spec. This is the language's **own** OT identifier — for Macedonian it remains `"MKD "` even though `language_tag` is currently `"sr"`. |
| `iso639_1`, `iso639_3` | `language-tags.json` | ISO 639-1 / ISO 639-3 codes. `null` when the standard has no assignment. |
| `confidence` | `language-tags.json` | `"high"` / `"medium"` / `"low"` — reliability of the ISO / BCP 47 / OT mapping. |
| `note` | `language-tags.json` | Free-text rationale when the mapping involved judgement (macrolanguages, dialect variants, provisional codes, OT tag collisions). `null` when no note is recorded. |

### `characters[]` — per-entry fields

| field | semantics |
| --- | --- |
| `character` | The Unicode character as a 1-element string. PUA codepoints typically render as a blank glyph in default fonts; use the `diagram` paths to see the intended shape in PT Sans/Serif Expert. |
| `codepoint` | 4-hex uppercase (`"0431"`, `"04D0"`, `"F510"`). |
| `decomposition` | Ordered array of hex codepoints this character decomposes to under the project's Paratype-aware NFD. Empty `[]` when the character is a structural modification (DESCENDER, HOOK, TAIL, STROKE, BAR, TICK, UPTURN, MIDDLE HOOK, VERTICAL STROKE, STROKE-AND-HOOK) with no standard combining-mark equivalent. The Paratype-shaped Cyrillic breve appears as `F6D1` (uppercase) / `F6D4` (lowercase). |
| `case` | `"uppercase"`, `"lowercase"`, or `"common"`. Only `U+02BC` (modifier apostrophe) currently uses `"common"`. |
| `description` | Unicode-style name — e.g. `"Cyrillic Capital Letter A With Breve"`. |
| `categories` *(per-language only)* | Subset of `["alphabet", "extended", "historic", "dialect", "consideration"]` — which `glyphs_list` block(s) of this language contain the codepoint. Multi-category when a codepoint appears in several blocks. |
| `diagram` | SVG and glyphplotter paths for Sans and Serif families — see [Path convention](#path-convention) below. |
| `variants` | Array of locl / style variants attached to this codepoint (may be empty). |
| `used_by` *(pan only)* | Sorted array of `name_eng` values — every in-scope language that includes this codepoint. |

### `variants[]` — per-entry fields

| field | semantics |
| --- | --- |
| `kind` | Currently always `"locl"`. (`"alternate"` is reserved for future support of `+`-only alternates that don't appear in `glyph-variants.md`.) |
| `language_tag` | BCP 47 tag whose OT locl this variant is keyed to. One of `bg`, `sr`, `ba`, `cv`. |
| `feature_locl_tag` | 4-letter OT feature tag that drives the substitution in the font: `BGR `, `SRB `, `BSH `, `CHU ` (with trailing space). |
| `style` | `"any"`, `"straight"`, or `"italic"` — maps to the `.str` / `.ita` suffix on the source `&`-marker token. `"any"` means no style qualifier. |
| `diagram` | `default → variant` SVG / plotter paths (see [Path convention](#path-convention)). |
| `used_by` *(pan only)* | Sorted list of `name_eng` that declare this variant. May contain more than one language — e.g. Serbian and Macedonian share the `sr` locl. |

## Path convention

Every `diagram.*.sans`, `diagram.*.serif`, and `diagram.plotter.*` value is a **forward-slash path relative to `cyrillic-reference/`** (the repo directory one level above this one). Paths never use `../` and never begin with `/` — the JSON carries pure suffixes that a consumer resolves against a base.

To resolve from a consumer:

- **Filesystem:** `/absolute/path/to/cyrillic-reference/` + the path value.
- **HTTP** (once published): deployed base URL ending in `cyrillic-reference/` + the path value.

Path shape:

```
svg/<Family>/<scope>/<identifier>.svg
glyphplotter/<Family>/<scope>/<identifier>.txt

<Family>     = "Sans" | "Serif"
<scope>      = "uc"       — uppercase character diagrams
             | "lc"       — lowercase character diagrams
             | "variants" — default → variant locl diagrams
<identifier> = <4-hex codepoint>                          when scope ∈ {uc, lc}
             | <4-hex codepoint> "." <ot locale>          when scope = "variants"
                                                              (ot locale ∈ {bg, sr, ba, cv})
```

Worked examples:

| JSON field | Resolved relative path |
| --- | --- |
| `diagram.svg.sans` for `А` (U+0410) | `svg/Sans/uc/0410.svg` |
| `diagram.svg.serif` for `й` (U+0439) | `svg/Serif/lc/0439.svg` |
| `diagram.plotter.sans` for the `sr` locl of `б` (U+0431) | `glyphplotter/Sans/variants/0431.sr.txt` |
| `diagram.svg.serif` for `Ē` (U+F510, PUA) | `svg/Serif/uc/F510.svg` |

Same codepoint, opposite case (`Й` / `й` = `0419` / `0439`) lives in separate files under `uc/` and `lc/`. The caseless modifier apostrophe `U+02BC` is filed under `lc/` for historical reasons — its `case` in the JSON is `"common"` but its diagram path resolves to `svg/<Family>/lc/02BC.svg`.

Variants in `pan-cyrillic.json` carry `used_by[]`; the same variants in `languages/<Name>.json` don't — in a per-language file the consumer is already this one language.

## Generator inputs and invariants

[`../tools/generate_json.py`](../tools/generate_json.py) reads `cyrillic-languages/` directly via the shared [`../tools/_catalog.py`](../tools/_catalog.py) core. It is **independent** of the Markdown tables and the SVG stage — all three generators are siblings and can run in any order.

| Input | Authoritative for |
| --- | --- |
| `../../cyrillic-languages/site/cyrillic/cyrillic_characters_lib.json` | Pan-Cyrillic deduplicated codepoint list: `character`, `codepoint`, `description`. Produced by `compile_languages.py` in the source repo. |
| `../../cyrillic-languages/library/cyrillic/base/<Name>.json` | Which codepoints a given language uses (via the tokenised `glyphs_list` blocks and their `type`); `alt_names_eng`, `name_rus`, `local`. Also the source for `&` / `+` variant markers. |
| `../../cyrillic-languages/library/cyrillic/cyrillic_library.json` | Registry of enabled languages, `code_pt`, `language_tag`. |
| [`../language-tags.json`](../language-tags.json) | BCP 47 / ISO / OT enrichment, `confidence`, `note`. |

`decomposition` is computed live per row by `_catalog.decompose`, which applies the Paratype-aware NFD (`… WITH <mark>` parsing → base + combining mark sequence, with the BREVE swap to U+F6D1 / U+F6D4). Structural composites (DESCENDER, HOOK, STROKE, …) return `[]` — their shape has no standard Unicode sequence.

Token filtering in `glyphs_list` (per `_catalog.extract_language_codepoints`):

- Tokens prefixed with `:` (explicit digraph marker) are dropped.
- Multi-character tokens without a known style suffix (`Гь`, `Ӷә`, `Ҭә`, …) are dropped — they represent digraphs/trigraphs that don't have their own codepoint.
- `!FXXX` / `!FXXXX` escape tokens resolve to the literal PUA codepoint.
- Style-qualified tokens (`г.ita`, `Д.str`) resolve to the base character's codepoint; the suffix becomes the variant's `style`.
- `+`-prefixed tokens introduce distinct codepoints into the language's `characters[]` (e.g. Bashkir's `Ғ ғ Ҙ ҙ Ҫ ҫ`). They do **not** currently become `variants[]` entries — only `&`-marker tokens do.
- `&`-prefixed tokens become `variants[]` entries on the codepoint they carry (`_catalog.aggregate_variants` collapses duplicates across languages into one row per (codepoint, locale, style) with a merged `used_by[]`).

**Invariant:** `codepoint ∈ lang.characters[].codepoint` ⇔ `lang ∈ pan[codepoint].used_by[]`. `used_by[]` at the pan level is computed from the same per-language extraction as the per-language output, so the two views agree by construction. A language that references a codepoint only inside a multi-char digraph token (e.g. Abkhazian's `Ӡʼ` → U+02BC) does **not** appear in `pan[02BC].used_by[]`.
